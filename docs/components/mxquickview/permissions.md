# Права доступа

У `mxQuickView` нет отдельной страницы в разделе **Компоненты** и нет собственного manager-права для UI.

## Что проверяется при рендере

- Доступ к ресурсу проверяется в процессоре:
  - `resource->checkPolicy('view')`
  - `resource->checkPolicy('load')`
  - `hasPermission('view')`
- Контекст запроса должен соответствовать `context_key` ресурса (при явной передаче `context`).
- Ресурс должен быть доступен по публикации: `published=1`, корректные `pub_date/unpub_date`, либо право `view_unpublished`.

Если у пользователя нет прав на просмотр ресурса, коннектор вернёт `Access denied`.

## Практика

- Отдельно настраивать права для страницы компонента не требуется.
- Доступ регулируется стандартными политиками MODX на ресурсы и контексты.
