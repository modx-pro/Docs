# Быстрый старт

Минимальные шаги, чтобы за 5 минут включить mxEditorJs в MODX 3.

## Требования

| Требование | Версия |
| --- | --- |
| MODX Revolution | 3.0.3+ |
| PHP | 8.2+ |

## Шаг 1: Установка

1. Установите пакет **mxEditorJs** через **Пакеты → Установщик** (загрузка transport-пакета или репозиторий).
2. Очистите кэш MODX.
3. Проверьте наличие системных настроек с префиксом `mxeditorjs` в **Настройки → Системные настройки**.

## Шаг 2: Выбор редактора

1. **Настройки → Системные настройки**
2. Найдите настройку **which_editor** (или отфильтруйте по «editor»).
3. В значении выберите **mxEditorJs**.
4. Сохраните.

## Шаг 3: Включение компонента

1. В системных настройках отфильтруйте по пространству имён **mxeditorjs**.
2. Убедитесь, что **mxeditorjs.enabled** = **Да** (по умолчанию включено).
3. При необходимости задайте **mxeditorjs.profile** (`default`, `minimal`, `blog`, `full`).

## Шаг 4: Проверка

1. Откройте любой ресурс в менеджере (создание или редактирование).
2. В поле контента должен отображаться блочный редактор Editor.js вместо TinyMCE/CKEditor.
3. Добавьте блок (заголовок, параграф, картинка), сохраните ресурс — контент сохраняется в формате Editor.js и выводится как HTML на фронте.

## Шаг 5: Template Variables (опционально)

Чтобы использовать mxEditorJs в TV:

1. Создайте или отредактируйте TV типа **Текст (многострочный)**.
2. В параметрах TV включите опцию **Использовать визуальный редактор** (richtext).
3. В системных настройках `which_editor` должен быть **mxEditorJs** — тогда в этом TV тоже будет блочный редактор.
