# Управление секциями

Страница в MigxPageConfigurator состоит из **секций** — крупных блоков (герой, преимущества, отзывы, баннер). Редактор даёт управлять не только содержимым секций, но и их **составом, порядком и видимостью** — через отдельную панель «Секции».

## Панель «Секции»

Откройте панель кнопкой в верхней панели редактора. В ней — список секций страницы в том же порядке, что на странице. У каждой секции — её название и набор управляющих значков:

- **⋮⋮** — перетащить, чтобы изменить **порядок**;
- **👁** — **показать / скрыть** секцию;
- **📌** — сделать секцию **статической** (см. ниже);
- **🔒** — секция **наследуется из типа** (клик по значку копирования переносит её в страницу).

**Клик по названию** секции прокручивает страницу к этой секции и подсвечивает её — удобно быстро найти нужный блок на длинной странице.

::: tip Изменения видны после «Обновить»
Перестановка, скрытие и переключение статичности сохраняются сразу, но точный итоговый вид страницы появляется после нажатия «Обновить страницу» (внизу панели).
:::

## Порядок секций

Порядок задаётся полем **позиции** каждой секции. Перетаскивание ⋮⋮ меняет позиции и сохраняет новый порядок. Секции с позицией `-1` всегда идут **вверху** (удобно «закрепить» шапку-баннер). Порядок в панели совпадает с порядком на странице.

Важно: **тип страниц задаёт набор секций, а каждая страница может переопределить их порядок и видимость** независимо. Поменяли порядок на одной странице — на других страницах того же типа он не изменится.

## Видимость

Значок 👁 скрывает секцию со страницы или показывает обратно. Скрытая секция остаётся в конфигурации (её содержимое не теряется) — просто не выводится. Это быстрый способ временно убрать блок, не удаляя его.

## Статические секции

Значок 📌 делает секцию **статической** — такие секции хранят контент в одном месте и одинаковы на всех страницах (баннер акции, блок «нам доверяют»). Правишь один раз — меняется везде. Подробно — в разделе [Статические секции](../migxpageconfigurator/static-sections) доки mpc.

## Наследование из типа

Если секция есть в **типе страниц**, но ещё не в самой странице, она показывается в панели под значком 🔒 — как **наследуемая**. Её содержимое выводится на странице, но правится оно у типа (то есть на всех страницах этого типа сразу).

Чтобы изменить секцию **только для этой страницы**, скопируйте её в страницу (значок копирования у наследуемой секции). После этого секция станет «своей», и правки будут локальными. Тот же выбор редактор предлагает, если вы пытаетесь добавить строку в список наследуемой секции (см. [Списки и строки](rows#списки-в-наследуемой-секции)).

## Массовые операции из типа

Вверху панели — кнопки для работы с набором секций целиком (когда страница наследует секции от типа):

- **＋ Дополнить из типа** — добавить секции типа, которых ещё нет у страницы (правки существующих сохраняются);
- **⟳ Перезаписать из типа** — заменить секции страницы секциями типа целиком (локальные правки теряются — с подтверждением);
- **🗑 Очистить** — удалить секции страницы; страница вернётся к наследованию из типа (с подтверждением).

Подробнее о копировании секций — в разделе [Копирование секций](../migxpageconfigurator/copy-sections) доки mpc.
