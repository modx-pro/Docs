# Системные настройки

Все настройки — с префиксом `mxboard.`, сгруппированы по областям (area). Задаются в **Система → Системные настройки** (фильтр по namespace `mxboard`) — с человеческими подписями и описаниями на русском и английском.

::: tip Обновление не меняет ваши значения
Резолверы добавляют недостающие настройки, но **не перезаписывают** уже существующие. Если в новой версии сменился дефолт (так было с `mxboard.group_admin_authority`: `0` → `1` в 2.8.0), на обновлённом сайте останется старое значение — правьте вручную.
:::

## Основные — область `mxboard_main`

| Настройка | По умолчанию | Что делает |
| --- | --- | --- |
| `mxboard.task_num_format` | `{y}{m}-{num}` | Шаблон человекочитаемого номера задачи (`num`). Плейсхолдеры: `{Y}`/`{y}` — год (4/2 знака), `{m}` — месяц, `{d}` — день, `{num}` — счётчик периода. Период сброса счётчика — по самому мелкому date-плейсхолдеру. |
| `mxboard.default_project` | `default` | Проект, используемый, когда клиент не указал `project` явно. |
| `mxboard.media_source` | `0` | ID медиа-источника MODX для вложений и полей-файлов. Заполняется резолвером при установке. `0` — источник по умолчанию. |
| `mxboard.upload_max_size` | `10485760` | Максимальный размер вложения, байт (`0` — без своего лимита). |
| `mxboard.upload_max_files` | `10` | Сколько файлов можно приложить за раз (`0` — без лимита). |
| `mxboard.upload_extensions` | `jpg,jpeg,png,…,json` | Разрешённые расширения через запятую (пусто — не ограничивать на стороне mxBoard). |
| `mxboard.sse_enabled` | `1` | Живой поток in-app уведомлений (SSE) в админку. Выключение гасит и генерацию уведомлений, и эндпоинт. |
| `mxboard.sse_lifetime` | `25` | Время жизни одного SSE-соединения, сек (клиент переподключается сам). Держится коротким ради shared-хостинга. |
| `mxboard.sse_poll_interval` | `3` | Интервал опроса новых уведомлений внутри соединения, сек. |

## API — область `mxboard_api`

| Настройка | По умолчанию | Что делает |
| --- | --- | --- |
| `mxboard.api_enabled` | `1` | REST-API для агентов включён. |
| `mxboard.mcp_enabled` | `1` | MCP-эндпоинт (JSON-RPC) для агентов включён. |

## Права — область `mxboard_rights`

| Настройка | По умолчанию | Что делает |
| --- | --- | --- |
| `mxboard.allow_self_close` | `0` | Разрешить закрывать карточку, где автор и исполнитель — один пользователь. По умолчанию запрещено (иначе агент сам себе ставит «сделано»). |
| `mxboard.wip_limit` | `0` | Сколько задач один исполнитель держит в работе одновременно (`0` — без лимита). |
| `mxboard.group_admin_authority` | `1` | Порог `authority` роли, при котором член группы отдела считается её менеджером (в MODX меньше `authority` = больше прав). `1` — менеджер это участник с ролью верхнего уровня, глобальный Super User для управления отделом не нужен. `0` — механизм выключен. См. [Модель прав](permissions). |
| `mxboard.kiosk_usergroups` | *(пусто)* | CSV имён групп MODX, чьих рядовых (не sudo) членов при входе в менеджер редиректить сразу на канбан. Пусто — выключено. См. [Интеграция → Киоск-доступ](integration). |

## ИИ-проверка — область `mxboard_ai`

Подробнее — [Типы задач → ИИ-проверка](types).

| Настройка | По умолчанию | Что делает |
| --- | --- | --- |
| `mxboard.ai_base_url` | *(пусто)* | Базовый URL OpenAI-совместимого эндпоинта (клиент добавит `/chat/completions`). Работает с mimo, OpenAI, DeepSeek, локальными. |
| `mxboard.ai_api_key` | *(пусто)* | Ключ доступа к провайдеру (Bearer). Пусто — проверка выключена (fail-open). |
| `mxboard.ai_model` | `mimo-v2.5` | Имя модели: `gpt-4o-mini`, `deepseek-chat` и т.п. |
| `mxboard.ai_check_mode` | `strict` | `strict` — неполную задачу создать нельзя; `soft` — предупреждение с «всё равно создать». |
| `mxboard.ai_check_prompt` | *(шаблон)* | Глобальный промпт-шаблон проверки. Тип может переопределить его своим `ai_prompt`. |

::: tip Секреты — не в репозиторий
`ai_api_key` и подобные значения проставляются в настройках конкретного сайта, а не в собираемом пакете.
:::
