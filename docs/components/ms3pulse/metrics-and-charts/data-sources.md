# Источники данных для графиков

Графики строятся по одному из пяти источников данных, получаемых с сервера (процессор `GetChartsData`).

## Список источников

| Источник | Ключ | Описание |
|----------|------|----------|
| Выручка по дням | `revenue_by_day` | Сумма заказов по дню/неделе/месяцу (зависит от группировки). |
| Заказы по дням | `orders_by_day` | Количество заказов по дню/неделе/месяцу. |
| Средний чек по дням | `avg_check_by_day` | Выручка ÷ количество заказов за каждый интервал. |
| Топ товаров | `top_products` | Товары, отсортированные по убыванию выручки (название, выручка, количество, маржа). |
| Заказы по статусам | `orders_by_status` | Количество заказов в каждом статусе (название статуса, количество, цвет). |

## Учёт фильтров

Все источники учитывают:

- **Период** («От» и «До»);
- **Статусы заказов** (если не выбраны — все статусы);
- **Группировка** (день/неделя/месяц) — для источников «по дням».

## Ограничение количества точек

В форме конструктора графиков задаётся **кол-во данных** (5, 10, 20, 50, 100):

- Для **выручка/заказы/средний чек по дням** — показываются последние N точек (по времени).
- Для **топ товаров** и **заказы по статусам** — первые N записей (топ-N товаров или все статусы до N).

Это помогает избежать перегруженных графиков при большом диапазоне дат или большом числе товаров.
