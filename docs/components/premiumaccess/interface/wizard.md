# Мастер доступа

![Мастер доступа PremiumAccess](/components/premiumaccess/screenshots/wizard.png)

**Мастер** (`/wizard`) за один шаг создаёт **тариф → группу → правила** на выбранные страницы. Если любой шаг не удался, откатывается вся операция.

Используйте мастер для **нового** платного раздела. Для одной страницы удобнее [Ресурсы](resources-and-blocks).

## Когда выбирать мастер

| Сценарий | Инструмент |
| --- | --- |
| Новый курс из нескольких статей + один тариф | **Мастер** |
| Закрыть одну существующую страницу | **Ресурсы** → привязка тарифа |
| Добавить PDF к уже проданному тарифу | **Правила** → правило `file` |
| Premium block внутри открытой статьи | **Premium-блоки** + маркер в content |

## Шаг 1: Продукт доступа

| Поле | Описание |
| --- | --- |
| **Название** | Отображается в кабинете и закрытом CTA |
| **Цена** | `price_amount` + `price_currency` — для CTA на витрине (не списание MS3) |
| **Срок** | `lifetime`, `period` (дней) или `fixed_date` — см. [Продукты и правила](products-and-rules) |
| **ms_product_id** | ID товара [MiniShop3](/components/minishop3/). **Обязателен** для покупки на витрине |

Товар MS3 создайте заранее: **MiniShop3 → Товары**. Цена и оплата берутся из MS3, не из полей тарифа PremiumAccess (поле «Цена» в тарифе — только для текста на закрытой странице).

## Шаг 2: Ресурсы

Выберите страницы MODX, которые войдут в группу. Для каждого ресурса мастер создаст правило `target_type=resource`.

Можно добавить несколько ресурсов одним тарифом: покупатель получает доступ ко всем страницам группы после одной оплаты.

## Шаг 3: Проверка и создание

Проверьте тариф, список ID страниц и нажмите **Создать**. Backend: `POST mgr/wizard/create-access`.

После успеха:

1. **Правила** — список правил с ID нового тарифа.
2. **Доступы** — пусто до первой покупки или ручной выдачи.
3. **Журнал** — записи о создании.

## Связь с miniShop3

```
Тариф PremiumAccess (id=1, ms_product_id=160)
    └── Правило: страница 2046
    └── Правило: страница 2047
Товар MS3 (id=160) — в корзине покупателя
```

После оплаты заказа (статус «Оплачен») компонент находит тариф по **ID товара MS3** и выдаёт доступ. Тогда пользователь откроет страницы 2046 и 2047 из примера выше.

## Права

Нужны права **`premiumaccess_manage_groups`** и **`premiumaccess_manage_rules`** (или `manage_products` для создания тарифа).
