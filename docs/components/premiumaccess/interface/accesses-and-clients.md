# Доступы, клиенты и промокоды

Вкладки **Доступы**, **Клиенты** и **Промокоды** нужны, когда доступ выдаётся **не через оплату** в miniShop3: вручную, по промокоду или для просмотра истории.

## Запись о доступе пользователя

Каждая выдача — строка в таблице `pa_user_accesses`. В менеджере это вкладка **Доступы**.

| Поле | Что означает |
| --- | --- |
| `user_id` | Пользователь сайта (web-контекст MODX) |
| `product_id` | Тариф |
| `source` | Откуда выдан: заказ MS3, вручную, промокод |
| `source_id` | ID заказа или активации промокода |
| `granted_at` | Когда выдали |
| `expires_at` | Когда истекает; пусто = бессрочно |
| `revoked_at` | Когда отозвали; пусто = не отозван |

### Статусы: Активен, Истёк, Отозван

Отдельного поля «статус» в базе нет — компонент считает его при каждом просмотре:

| Статус | Когда показывается |
| --- | --- |
| **Активен** | Не отозван и срок не истёк (или бессрочный) |
| **Истёк** | Дата окончания в прошлом, доступ не отзывали |
| **Отозван** | Администратор или система отозвали доступ |

### Доступ после оплаты в miniShop3

После оплаты запись создаёт плагин `premiumaccess_order_access`.

При **отмене или возврате** заказа отзывается только доступ, выданный **по этому заказу**. Ручная выдача и доступ по промокоду при отмене чужого заказа **не затрагиваются**.

## Вкладка «Доступы» {#вкладка-доступы}

![Доступы PremiumAccess](/components/premiumaccess/screenshots/accesses.png)

Список всех выдач с фильтрами и постраничной навигацией.

| Действие | Когда использовать |
| --- | --- |
| **Выдача** | Дать доступ B2B, тестовому аккаунту, после оплаты вне MS3 |
| **Отзыв** | Закрыть доступ досрочно |
| **Продление** | Сдвинуть дату окончания на N дней |
| **Редактирование дат** | Исправить дату выдачи или окончания |

### Выдача вручную

1. Укажите пользователя по ID или email (если email новый — создастся web-пользователь).
2. Выберите тариф и при необходимости дату окончания.
3. Сохраните.

Повторная выдача того же тарифа тому же пользователю не создаёт дубликат и не шлёт повторное уведомление.

### Отзыв

Доступ помечается как отозванный. Кэш отказов на витрине сбрасывается — закрытый контент снова не показывается этому пользователю.

### Продление

К текущей дате окончания добавляются указанные дни. Для бессрочного доступа используйте редактирование дат.

## Вкладка «Клиенты» {#вкладка-клиенты}

![Клиенты PremiumAccess](/components/premiumaccess/screenshots/clients.png)

Список пользователей, у которых есть **хотя бы одна** выдача доступа.

## Промокоды {#промокоды}

![Промокоды PremiumAccess](/components/premiumaccess/screenshots/promo-codes.png)

Вкладка **Промокоды**. Таблица `pa_promo_codes`.

| Поле | Описание |
| --- | --- |
| `code` | Текст кода; если оставить пустым при создании — сгенерируется (префикс `PA`) |
| `product_id` | Какой тариф выдать при активации |
| `max_uses` | Сколько раз можно активировать; пусто = без лимита |
| `uses_count` | Сколько раз уже активировали |
| `expires_at` | До какой даты действует сам код |
| `status` | active / disabled |

### Активация на сайте

Сниппет [PremiumAccessPromoRedeem](../snippets/PremiumAccessPromoRedeem):

1. Пользователь **авторизован** на сайте.
2. Вводит код в форме на странице.
3. При успехе получает доступ к выбранному тарифу; событие попадает в **Журнал**.

Не более 10 попыток в минуту на одного пользователя (защита от перебора).

### Один код — один пользователь

Один и тот же пользователь не может активировать один промокод дважды.

## Журнал

Вкладка **Журнал** — история выдач, отзывов, активаций промокодов, истечений срока.

| Событие | Что произошло |
| --- | --- |
| `access_granted` | Выдали доступ |
| `access_revoked` | Отозвали |
| `access_expired` | Истёк срок (cron) |
| `promo_redeem` | Активировали промокод |
| `auto_renewal_reminder` | Отправили напоминание о продлении |
| `access_check` | Проверка на витрине (если включено логирование) |

Фильтры по дате, пользователю и тарифу.
