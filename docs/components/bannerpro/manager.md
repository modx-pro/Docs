# Админка

Откройте **Компоненты → BannerPro**. Контроллер `BannerProIndexManagerController` монтирует Vue 3-приложение в `#bannerpro-mgr-app` и передаёт конфиг в `window.bannerproConfig`.

Интерфейс требует **VueTools**. Без него страница покажет предупреждение и не загрузит PrimeVue-компоненты.

## Права доступа

| Permission | Что разрешает |
| --- | --- |
| `view` | Открывает компонент в менеджере |
| `bannerpro_save` | Создаёт и редактирует баннеры, позиции, связи, порядок и вес |
| `bannerpro_remove` | Удаляет баннеры и позиции |
| `bannerpro_stats` | Открывает статистику, графики, экспорт CSV и рефереры |

Без `bannerpro_save` кнопки создания и редактирования скрыты. Connector вернёт HTTP 403 при прямой попытке изменить данные.

## Вкладка «Баннеры»

Список поддерживает серверную пагинацию, сортировку по колонкам, поиск по названию и описанию, фильтр по позиции.

| Поле | Что хранит |
| --- | --- |
| Название | Внутреннее имя баннера |
| Тип | `image` или `html` |
| URL | `[[~id]]`, внешний адрес или URL с плейсхолдерами |
| Источник файлов | Media Source для изображения |
| Активен | Флаг вывода на сайте |
| Изображение | Путь к файлу для `type=image` |
| HTML | Код баннера для `type=html` |
| Описание | Текст для `alt`, `title` или собственного чанка |
| Начало / Конец | Период показа |
| Лимит кликов / показов | `max_clicks`, `max_impressions`, `0` означает без лимита |
| Позиции | Одна или несколько рекламных позиций |
| Товар MiniShop3 | Опциональная привязка к `msProduct` |

Колонка **URL** показывает ресурс MODX как `(id) pagetitle`. Админка не выводит сырой HTML в колонке.

## Поле «Товар MiniShop3»

Поле появляется в форме баннера и ищет ресурсы `msProduct`.

| Поведение | Детали |
| --- | --- |
| Поиск | `resource_getlist`, фильтр по `pagetitle` и `class_key` |
| Лимит | До 200 товаров за запрос |
| Редактирование | Привязанный товар остаётся в списке |
| Подставить URL | Пишет `[[~{id}]]` в поле URL |
| Отвязать | Сбрасывает `product_id` |

Пикер не загружает весь каталог. Без фильтра он показывает первые 200 опубликованных товаров.

## Вкладка «Позиции»

Позиция задаёт логический слот на сайте: `sidebar`, `header`, `shop-product-sidebar`.

| Действие | Что делает |
| --- | --- |
| Создать | Добавляет позицию по имени |
| Редактировать | Открывает имя и список баннеров в позиции |
| Удалить | Удаляет позицию и связи с баннерами |
| Добавить баннер | Привязывает существующий баннер к позиции |
| Изменить порядок | Обновляет `idx` через drag-and-drop |
| Задать вес | Обновляет `weight` для `sortby=weighted` |

В чанке используйте плейсхолдер `adposition`. Это ID связи `bannerpro_ads_positions`, а не ID баннера.

## Вкладка «Статистика»

Вкладка доступна при праве `bannerpro_stats`.

| Блок | Что показывает |
| --- | --- |
| Период | Всё время, сегодня, текущий месяц, прошлый месяц, текущий год |
| Фильтр позиции | Все позиции или одна позиция |
| KPI | Клики, показы, CTR, заказы |
| Обзор | Топ баннеров с кликами, показами, заказами и CTR |
| Клики | График по баннерам |
| Рефереры | Таблица referrer + клики |
| Экспорт CSV | Сводка статистики |

Клики считаются один раз в сутки с одного IP на пару баннер + позиция. Показы работают только при включённом `bannerpro_track_impressions`.

## Типичный рабочий процесс

1. Создайте позицию `sidebar`.
2. Создайте баннер и привяжите его к `sidebar`.
3. В шаблоне вызовите `BannerPro` с `&positionName=`sidebar``.
4. Кликните по баннеру на сайте.
5. Откройте **Статистика** и проверьте клик.
