# Админка

Откройте **Компоненты → BannerPro**. Контроллер `BannerProIndexManagerController` монтирует Vue 3-приложение в `#bannerpro-mgr-app` и передаёт конфиг в `window.bannerproConfig`.

Нужен **VueTools**. Без него страница покажет предупреждение и не загрузит PrimeVue-компоненты.

Вкладки: **Баннеры**, **Позиции**, **Статистика**, **Журнал** (при `bannerpro_save`), **Настройки** (UTM, webhook, A/B).

## Права доступа

| Permission | Что разрешает |
| --- | --- |
| `view` | Открывает компонент в менеджере |
| `bannerpro_save` | Создаёт и редактирует баннеры, позиции, связи, порядок и вес |
| `bannerpro_remove` | Удаляет баннеры и позиции |
| `bannerpro_stats` | Открывает статистику, графики, экспорт CSV и рефереры |

Без `bannerpro_save` кнопки создания и редактирования скрыты. Connector вернёт HTTP 403 при прямой попытке изменить данные.

### Режим «только статистика»

Если у пользователя есть `bannerpro_stats`, но нет `bannerpro_save` и `bannerpro_remove`, открывается только вкладка **Статистика**. Вкладки «Баннеры» и «Позиции» скрыты.

| save | remove | stats | UI |
| --- | --- | --- | --- |
| 1 | * | * | Полный доступ |
| 0 | 0 | 1 | Только «Статистика» |

## Вкладка «Баннеры»

Список поддерживает серверную пагинацию (10 / 20 / 50), сортировку, поиск по названию и описанию.

Фильтры: **активен**, **тип** (image/html), **товар**, **категория MS3**, **метка**, **позиция**.

- **Создать из шаблона**: готовые баннеры из `bannerpro_ad_templates`
- **Массовые действия** (чекбоксы): включить, выключить, удалить, назначить позиции
- **Дублировать**: копия с теми же позициями, `idx` и весом. К названию добавляется « (копия)»
- Колонки **Клики**, **Заказы** (`conversions`, атрибуция MS3)

<!-- SCREENSHOT: Список баннеров с фильтрами и массовыми действиями -->
<!-- ![Вкладка «Баннеры»](/components/bannerpro/screenshots/manager-ads-list.png) -->

| Поле | Что хранит |
| --- | --- |
| Название | Внутреннее имя баннера |
| Тип | `image` или `html` |
| URL | `[[~id]]`, внешний адрес или URL с плейсхолдерами |
| Источник файлов | Media Source для изображения |
| Активен | Флаг вывода на сайте |
| Изображение | Путь к файлу для `type=image` |
| HTML | Код баннера для `type=html`. При URL оборачивается в ссылку клика |
| Описание | Текст для `alt`, `title` или собственного чанка |
| Начало / Конец | Период показа |
| Лимит кликов / показов | `max_clicks`, `max_impressions`, `0` означает без лимита |
| Расписание показа | Дни недели и часы (`show_hours`) |
| Страница / раздел | Таргетинг: ресурс или дочерние страницы раздела |
| Метки | Теги для фильтра `&tags=` в сниппете |
| Позиции | Одна или несколько рекламных позиций |
| Товар MiniShop3 | Привязка к `msProduct` |
| Категория MiniShop3 | Баннер для товаров в категории. Взаимоисключимо с товаром |

Колонка **URL** показывает ресурс MODX как `(id) pagetitle`. Для HTML-баннеров доступно **превью** в iframe-песочнице.

## Поле «Товар MiniShop3»

| Поведение | Детали |
| --- | --- |
| Поиск | `resource_getlist`, фильтр по `pagetitle` и `class_key` |
| Лимит | До 200 товаров за запрос |
| Редактирование | Привязанный товар остаётся в списке |
| Подставить URL | Пишет `[[~{id}]]` в поле URL |
| Отвязать | Сбрасывает `product_id` |

Пикер не загружает весь каталог. Без фильтра он показывает первые 200 опубликованных товаров.

## Вкладка «Позиции»

Позиция задаёт логический слот на сайте: `sidebar`, `header`, `shop-product-sidebar`.

| Действие | Что делает |
| --- | --- |
| Создать | Добавляет позицию по имени и опционально контексту |
| Редактировать | Имя, `context_key` и список баннеров в позиции |
| Дублировать | Копия позиции с теми же баннерами, порядком и весом |
| Удалить | Удаляет позицию и связи с баннерами |
| Изменить порядок | Обновляет `idx` через drag-and-drop |
| Задать вес | Обновляет `weight` для `sortby=weighted` |

**Контекст** (`context_key`): ключ MODX (`web`, `mgr`, …). Пустое поле: позиция во всех контекстах. На сайте сниппет фильтрует по текущему контексту MODX. Подробнее — в разделе [Интеграция](integration#контекст-modx).

В чанке подставляйте плейсхолдер `adposition`: ID связи `bannerpro_ads_positions`, не ID баннера.

## Вкладка «Статистика»

Вкладка доступна при праве `bannerpro_stats`.

| Блок | Что показывает |
| --- | --- |
| Период | Всё время, сегодня, последние 7 дней, эта/прошлая неделя, текущий/прошлый месяц, текущий год |
| Фильтр позиции | Все позиции или одна позиция |
| Сравнение | Переключатель «Сравнить с прошлой неделей»: два ряда на графике и процент изменения на карточках |
| Сводка | Клики, показы, CTR, заказы |
| Воронка | Показ → клик → заказ за период |
| Обзор | Топ баннеров с серверной пагинацией (10 / 20 / 50) |
| Клики | График по баннерам |
| Рефереры | Таблица источников перехода и кликов |
| Экспорт CSV | Сводка статистики |

На графиках по дням отображаются линии **conversions** (заказы MS3).

Клики считаются один раз в сутки с одного IP на пару баннер + позиция. Показы работают только при включённом `bannerpro_track_impressions`.

## Вкладка «Журнал»

Доступна пользователям с `bannerpro_save`. Журнал изменений: дата, пользователь, действие (`create`, `update`, `bulk_enable`, …), объект (баннер/позиция), ID, краткое описание. Фильтры по типу объекта и действию.

## Вкладка «Настройки»

Единая форма для интеграций:

- **UTM** при клике (`bannerpro_utm_*`)
- **Webhook** клик и показ (`bannerpro_webhook_url`, `bannerpro_webhook_impression_url`, `bannerpro_webhook_secret`)
- **A/B-деление** (`bannerpro_ab_ttl`)

Сохранение через connector actions `settings_integrations_get` / `settings_integrations_update`. Те же ключи доступны в **Система → Настройки системы** → namespace `bannerpro`.

## Типичный рабочий процесс

1. Создайте позицию `sidebar`.
2. Создайте баннер и привяжите его к `sidebar`.
3. В шаблоне вызовите `BannerPro` с `positionName=sidebar`.
4. Кликните по баннеру на сайте.
5. Откройте **Статистика** и проверьте клик.
