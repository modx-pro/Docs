# Админка в MODX

Раздел **Extras → Почта России** в панели управления MODX: тест расчёта без витрины, журнал запросов к API, справочник кодов объектов и кнопка очистки кэша. Интерфейс на **Vue 3** и **PrimeVue 4**.

## Требования

Нужен установленный **[VueTools](https://docs.modx.pro/components/vuetools/)** (Vue 3 + PrimeVue). Без VueTools при открытии пункта меню показывается предупреждение с подсказкой по установке. На **сайт магазина** и виджет оформления заказа VueTools **не влияет**.

## Кнопка «Очистить кэш»

Вверху страницы. Удаляет закэшированные ответы API из `core/cache/msrussianpost/`.

Нажимайте после изменения [системных настроек](settings) (индекс отправителя, коды объектов, режим API и т.д.), при переключении `tariff` ↔ `otpravka` и если подозреваете устаревшие тарифы. После успешной очистки обычно показывается Toast «Кэш очищен».

## Таб «Тестовый расчёт»

![Тестовый расчёт](/components/msrussianpost/screenshots/admin-ui-test-calculation.png)

![Тестовый расчёт](/components/msrussianpost/screenshots/admin-ui-test-calculation-2.png)

Проверка тарифов **без** оформления заказа на витрине.

| Поле | Смысл | Откуда берётся по умолчанию |
|------|--------|------------------------------|
| Индекс отправителя | Откуда везём, 6 цифр | `msrussianpost_sender_index` |
| Индекс получателя | Куда везём, 6 цифр | ввод вручную (обязательно) |
| Вес (г) | Вес отправления | `msrussianpost_default_weight` |
| Коды объектов | Типы отправлений через запятую | `msrussianpost_object_codes` |

Нажмите **Рассчитать**. В таблице результатов: **услуга**, **стоимость**, **срок** (дни), **API** — источник строки: публичный `tariff.pochta.ru` или персональный `otpravka.pochta.ru`.

**Если таблица пуста** — для заданных индексов/кодов/веса Почта не вернула тарифы; проверьте параметры и журнал. **Если в режиме otpravka в колонке API указан публичный** — сработал [fallback](integration#api-fallback) (ошибка/лимит персонального API или не заданы ключи).

## Таб «Журнал запросов»

![Журнал запросов](/components/msrussianpost/screenshots/admin-ui-request-log.png)

Доступен, когда в настройках включено **`msrussianpost_log_enabled`**. Показывает время запроса, индексы, вес, стоимость из ответа, длительность (мс) и использованный API (`tariff` / `otpravka`). Кнопки **Очистить журнал** и **Обновить**.

Если логирование выключено или файл журнала пуст, на странице выводится соответствующее сообщение.

## Таб «Справочник объектов»

![Справочник объектов](/components/msrussianpost/screenshots/admin-ui-object-dictionary.png)

Подгружает описания **только тех кодов**, что заданы в **`msrussianpost_object_codes`**: для каждого кода запрос к публичному API (`dictionary/object/list`). Так можно убедиться, что коды существуют и совпадают с нужными услугами (название, поля **mailtype** / **mailctg** / **directctg** в терминах тарификатора).

1. Сохраните коды в системных настройках.
2. Нажмите **Загрузить справочник**.
3. Поле поиска фильтрует строки на клиенте.
4. После смены списка кодов снова загрузите справочник при необходимости.

## Типовой порядок работы

1. Заполните **`msrussianpost_sender_index`** и **`msrussianpost_object_codes`**, при необходимости режим API и ключи otpravka — см. [Системные настройки](settings).
2. На вкладке **Справочник объектов** проверьте коды.
3. На **Тестовом расчёте** проверьте маршрут и вес.
4. При диагностике включите **`msrussianpost_log_enabled`**, воспроизведите сценарий, смотрите **Журнал**, затем выключите логирование.
5. После любых правок настроек не забывайте **Очистить кэш** на этой же странице.
